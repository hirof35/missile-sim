# Missile Guidance Simulation (Proportional Navigation)

HTML5 Canvas と JavaScript（Node.js環境対応可）で実装された、ミサイルの誘導アルゴリズム（比例航法）の視覚シミュレーターです。
ターゲット（目標物）の動きを先読みし、効率的な衝突コース（コリジョンコース）を自動生成して追尾する様子をリアルタイムに観察できます。

## 🚀 デモの動かし方

### 1. ブラウザで直接開く場合（最も簡単）
1. `index.html` をローカルに保存します。
2. 保存したファイルをブラウザ（Chrome, Edge, Firefoxなど）にドラッグ＆ドロップします。

### 2. Node.js（静的サーバー）環境で動かす場合
`http-server` などを用いてローカルサーバーを立ち上げて実行する場合：

```bash
# 静的サーバーのインストール（未導入の場合）
npm install -g http-server

# サーバーの起動
http-server .
起動後、ターミナルに表示される http://localhost:8080 にアクセスしてください。🎮 操作方法ターゲット（赤色の円）: マウスのカーソルに追従して移動します。マウスを激しく動かしてミサイルを翻弄してみてください。ミサイル（青色の円）: 画面下部から発射され、比例航法アルゴリズムに基づいてターゲットを自動追尾します。リセットボタン: シミュレーションを初期状態に戻し、ミサイルを再発射します。軌跡の描画: 赤と青の薄い線は、それぞれの移動経路（トラジェトリ）を示しています。白い薄い線は、毎フレームのミサイルからターゲットへの「視線（Line of Sight）」を表します。🛠 導入されているアルゴリズム：比例航法（Proportional Navigation）本シミュレーターでは、実際の対空ミサイルや誘導兵器でも広く使われている比例航法（PN: Proportional Navigation）を採用しています。ターゲットを単に後ろから追いかける「追従法（Pure Pursuit）」とは異なり、比例航法は「自分から見たターゲットの方角（視線角）の変化速度に比例して、自分の旋回量を決める」という手法です。数式的なアプローチミサイルの旋回指令（進路の変更速度 $\dot{\gamma}$）は以下の式で決定されます。$$\dot{\gamma} = N \times \dot{\lambda}$$$\dot{\lambda}$ : 視線角速度（ターゲットの方角が1フレームでどれだけズレたか）$N$ : ナビゲーション定数（本システムでは 4.0 を採用。一般的に 3.0 〜 5.0 が最適とされる）これにより、ミサイルはターゲットの未来位置を「先読み」するように回り込み、最短ルートで命中させることができます。📦 コードのカスタマイズindex.html 内の以下の定数を書き換えることで、シミュレーションの挙動を調整できます。N: ナビゲーション定数を小さく（例: 1.0）すると追従法に近くなり、横移動するターゲットに追いつけなくなります。大きくしすぎると挙動が過敏になります。MISSILE_SPEED / TARGET_SPEED: それぞれの移動速度を調整できます。ターゲットをミサイルより速くするとどうなるか、といった実験が可能です。📝 ライセンスMIT License
